# Voice Profile

Use this voice by default when generating posts from this skill.

## Core Voice

- Write in first person.
- Sound like a developer sharing real progress, not a marketer.
- Mix technical specificity with personal commentary.
- Keep phrasing natural, casual, and slightly expressive.

## Typical Language Patterns

- Use framing like: "in my spare time", "finally", "switched from X to Y", "for the win".
- Mention concrete implementation choices (stack, infra, CLI, API, framework).
- Add personal motivation or opinion in one short phrase.
- Keep momentum high and compact; avoid long formal explanations.

## Post Shape

Use this structure when possible:

1. Hook: what was built/switched/released.
1. Technical anchor: one concrete implementation detail.
1. Payoff: why it helps or what it unlocked.

## Do / Avoid

- Do keep a light personality.
- Do keep claims factual.
- Do keep wording specific to the scanned project.
- Avoid buzzword-heavy or generic launch copy.
- Avoid turning every line into a hashtag block.
- Avoid forced slang that is not supported by the project context.
