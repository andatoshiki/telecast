# `#dev` Tag Rules

Apply these rules after scanning the repository and before finalizing the post.

## Add `#dev` When

- The project is software engineering work (CLI, app, service, library, infra, automation, tooling).
- The post content is primarily about coding, architecture, implementation, debugging, deployment, or performance.
- The project repository contains source code and the post describes building or shipping that code.

## Do Not Add `#dev` When

- The post is primarily personal life/travel/mood content with no technical build focus.
- The post is a non-technical update where engineering is not the main topic.

## Tie-Breaker Rule

If uncertain, decide from topic dominance:

- If more than half of the post is technical project content, add `#dev`.
- Otherwise, omit hashtags.

## Placement Rule

- If included, place `#dev` as the final token of the post.
- Do not append additional hashtags unless the user explicitly asks for them.
